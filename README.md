# 규칙
  ## git 규칙
    - 각자의 branch에서 작업
    - 한 단위 끝날 때마다 commit
    - 하루 한 번 push
    - 팀장이 pull request 확인 후 merge
  ### commit message 작성법
    - 첫번째 줄에 수정한 것에 대한 요약
    - 두번째 줄 비움
    - 세번째 줄에 상세한 작업 내역 작성
      1. 완료한 작업
      2. 진행중인 작업 (있다면)
      3. 어떠한 기능을 구현했는지 작성
  ### PR(pull request) 방법
    - 제목에 PR날짜, commit들을 통합해서 요약
    - 내용 생략
